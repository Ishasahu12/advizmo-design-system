Component Weight System

Visual hierarchy rules for recommendation importance levels

1. Component Weight Overview

Six weight levels determine how recommendation types appear visually, from most prominent to least.

1
Hero Module

Largest. Dark gradient, maximum visual weight.

2
Primary Recommendation Card

Large semantic gradient, important action.

3
Secondary Recommendation Card

Compact, lower emphasis.

4
Review Card

Expandable, designed for reviewing.

5
Inline Status Banner

Small, minimal height.

6
Success Card

Temporary, auto-dismiss.

2. Hero Module

Rules

  • Used ONLY for Financial Setup Stepper
  • Appears ONLY in hero section
  • Maximum one on screen
  • Dark gradient background (#1c1c1e → #2c2c2e)
  • Replaces "Things to Fix" while onboarding incomplete
  • Disappears permanently when setup reaches 100%

6. Inline Status Banner

Rules

  • Small component, minimal height (40–48px)
  • For status information, not actions
  • No CTA or very subtle
  • Can appear anywhere inline

7. Success Card

Rules

  • Temporary confirmation
  • Appears after completing an action
  • Auto-dismisses after 2–3 seconds
  • Minimal visual weight, green surface
  • Subtle entrance animation (fade in + slide up)
