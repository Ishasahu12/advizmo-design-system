Section 9
Edge Cases
25+ real-world scenarios and how components handle them
1. Account name longer than 3 lines
Vanguard Total Stock Market Index Fund Admiral Shares (VTIAX) — 63 characters
Stacked: max-height 3.9em with overflow hidden. Split: -webkit-line-clamp 2 with ellipsis. Compact: single line ellipsis.
2. Balance exceeds $99,999,999.99
Displayed as $100,000,000.00+ with tooltip
Balance field caps display at "$100M+" with info icon. Full balance shown on long-press tooltip.
3. Transfer amount exceeds balance
User enters $500 but only $212.40 available
Red border on input, inline error: "Insufficient funds. Available: $212.40". Move button disabled.
4. Same source and destination
User tries to move money to the same account
Banner error: "Cannot transfer to the same account". Destination card shows red border with lock icon.
5. Duplicate destination in same move
Chase Checking added twice in multi-destination move
Amber warning badge: "Already added". Destination card gets amber border. Amounts shown side-by-side in review.
6. All destinations unsupported
Every connected account has compliance restrictions
Empty state with lock icon. Message: "No available destinations". CTA: "Connect a supported account".
7. Source account frozen
Bank has placed a hold on the source account
Source card shows lock overlay with gray styling. Error banner: "Account frozen — Contact your bank". All destinations disabled.
8. Source connection expired mid-transfer
Plaid token expires while user is entering amounts
Source card switches to "Reconnect" state. All destination inputs disabled. Warning banner: "Connection expired. Please reconnect."
9. Market closes during entry
User starts entering a transfer at 3:55 PM, market closes at 4:00 PM
Info banner appears: "Market closed. Transfer will execute at market open." Investment destination gets clock icon.
10. Bank returns error after submission
ACH network rejects the transfer post-submission
Destination switches to Error state with red X. Error message from bank displayed. Retry button shown.
11. Network timeout during transfer
User's connection drops during processing
Processing spinner continues. After 30s: "Taking longer than usual..." After 60s: "Check status in Activity". No duplicate submission.
12. User switches source mid-flow
User changes source after entering destination amounts
All destination amounts re-validated against new source balance. Insufficient funds flagged if needed. Confirmation toast: "Source updated".
13. Amount has decimal precision issues
User enters 250.999 or 100.001
Auto-rounded to 2 decimal places on blur. Helper text: "Rounded to $251.00". No silent precision loss.
14. Currency conversion needed
Source in USD, destination in EUR
Banner: "Currency conversion required". Shows estimated rate and converted amount. Fee disclosure inline.
15. International transfer restrictions
Destination account is in a restricted country
Destination card shows red "Unsupported" badge. Banner: "International transfers not supported to this region."
16. Daily transfer limit reached
User has sent $9,500 today, trying to send $1,000 more
Red banner: "Daily limit: $10,000. Remaining: $500". Input caps at $500. Limit exceeded message if user types more.
17. Monthly transfer limit reached
Regulatory limit on outbound transfers hit
Red banner: "Monthly limit: $25,000. Remaining: $0". All destination inputs disabled. Suggestion: "Try again next month."
18. Account requires 2FA
Destination account has additional verification
Destination card shows shield icon. "Verify" button triggers 2FA flow. Transfer blocked until verified.
19. Beneficiary name mismatch
Destination account name doesn't match registered beneficiary
Warning banner: "Beneficiary mismatch detected". Suggestion: "Verify account ownership before proceeding."
20. Routing number invalid
User manually enters incorrect routing number
Inline error: "Invalid routing number". Red border on input. Suggestion: "Check the 9-digit number on your checks."
21. Account number invalid
Entered account number fails validation checksum
Inline error: "Invalid account number". Red border. No partial match disclosure for security.
22. Minimum balance requirement
Source account requires $500 minimum, transfer would drop below
Warning: "Transfer will drop balance below $500 minimum". Suggests keeping $500 buffer. Allows override with confirmation.
23. Overdraft protection triggered
Transfer amount would trigger overdraft on source
Banner: "This transfer will use overdraft protection". Shows overdraft fee if applicable. Requires explicit confirmation.
24. Multiple moves queued simultaneously
User submits 3 moves at once from different screens
Queue indicator in footer: "3 moves queued". Activity badge shows count. Each move shows independent status.
25. User receives push notification during flow
Bank sends fraud alert while user is entering amounts
Notification banner appears at top. Flow state preserved. User can dismiss or act on notification without losing progress.
26. App backgrounded during transfer
User switches to another app during processing
Transfer continues server-side. On return: status updated. If failed: error state shown. No duplicate processing.
27. Split destination amounts don't sum correctly
User allocates $300 + $200 + $200 = $700 but source has $650
Summary validation: "Total exceeds source balance by $50". Individual insufficient errors on affected destinations.
28. Bank holiday affects settlement
User submits transfer on Friday evening, Monday is a holiday
Info banner: "Bank holiday — Settlement delayed to Tuesday". Shows estimated arrival date. No false urgency.
