Design System Reference

Component specifications, animation tokens, and usage guidelines for all loading and empty states.

Skeleton Components
KPI Skeleton
24px height bar for headline numbers
Text Skeleton
14px height bar for body text
Account Skeleton
32px circle for avatars and icons
Chart Skeleton
120px height rectangle with gradient
List Skeleton
3 rows of circle + bar + bar
Animation Tokens
Pulse (Skeleton)
Duration: 1.5s | Easing: ease-in-out | Loop: infinite
From: #e5e7eb | To: #f3f4f6
Spin (Loader)
Duration: 0.8s | Easing: linear | Loop: infinite
Dot Pulse
Duration: 1.4s | Stagger: 0.2s | Loop: infinite
Progress Fill
Duration: 3s | Easing: ease-in-out | Loop: infinite
Fade In
Duration: 0.4s | From: opacity 0, translateY 8px
Color Tokens
--blue: #0a84ff
Primary actions, loading spinners, links
--green: #166f47
Success states, health scores, positive values
--amber: #c2790a
Warnings, attention needed, removal states
--red: #b91c1c
Critical alerts, high priority fixes
--card: #ffffff
Card backgrounds, elevated surfaces
--text: #1c1c1e
Primary text, headings, labels
--text2: #8e8e93
Secondary text, placeholders, metadata
--bg: #f2f2f7
Page background, iOS system background
Banner Components
Blue banner — loading, connecting, syncing
Amber banner — warnings, removal, caution
Green banner — success, completion, synced
Banners use 10px 14px padding, 10px border-radius, and 13px font-weight 600. Always include an icon or spinner on the left.
CTA Button Variants
Connect account (Blue)
Open account (Green)
Connect accounts (Amber)
Cancel (Text link)
Buttons: 14px padding, 12px border-radius, 15px font-weight 600, white text. Always full-width in empty states.
Empty State Pattern
1. Illustration — 80px circle with colored background, 36px SVG icon centered
2. Title — 17px, font-weight 700, centered, 4px bottom margin
3. Description — 13px, #8e8e93, centered, 1.5 line-height, 8px bottom margin
4. Primary CTA — Full-width button, colored background, 16px top margin
5. Meta text — 11px, #8e8e93, centered, 8px top margin (optional)
6. Secondary CTA — Text link below button (optional)
Container: 24px vertical padding, 16px horizontal padding inside card
Typography Scale
Section Title 22px / 700
Welcome 20px / 700
Empty Title 17px / 700
Card Title 15px / 600
Body / Account Name 14px / 600
Caption / Banner 13px / 400
Meta / Account Sub 12px / 400
KPI Label / Timestamp 11px / 400 uppercase
Spacing Reference
4px — Tight spacing: between icon and text in account rows
8px — Compact spacing: between skeleton bars, icon gaps
12px — Default spacing: card padding, row gaps, element gaps
16px — Comfortable spacing: section headers margin, card padding
20px — Section spacing: between section header and card
24px — Page padding: horizontal padding for phone body
Loading State Guidelines
Never use full-screen spinners. Always show skeletons that match the final layout.
Initial load: Show all skeleton sections simultaneously. Banner at top indicates loading state.
Partial refresh: Only skeleton-ize sections that are being updated. Keep loaded content visible.
Account operations: Use colored banners (blue for sync, amber for removal) with spinner.
Progress indication: Use progress bars for quantifiable operations, dot pulses for indeterminate waits.
Progressive reveal: Sections should appear as data loads, not all at once.
Cancel option: Always provide cancel link for operations longer than 5 seconds.
Empty State Guidelines
Every empty state should help the user understand what to do next.
Illustration: Use simple line art, 36px, with colored background circle. Match illustration color to section purpose.
Title: Be specific about what's missing. "Connect your first account" beats "No accounts".
Description: One sentence explaining the value. "See all your money in one place".
Primary CTA: One clear action. Use color matching the section (blue for general, green for investment, amber for spending).
Context: Show surrounding populated sections when possible to give context.
No CTA states: Some sections (News, Recurring) don't need CTAs when data simply hasn't been detected yet.
Time estimates: Add "Takes about 2 minutes" for actions that require user effort.
