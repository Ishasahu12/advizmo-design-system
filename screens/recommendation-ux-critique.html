Section 1 — UX Critique

25 issues

Complete audit of the recommendation card experience across hierarchy, clarity, interaction, trust, scalability, and accessibility dimensions.

# Category Severity Current State Proposed Fix UX Reason
1 Hierarchy Critical Warning triangle icon creates anxiety Use helpful icons — checkmarks, lightbulbs, info cues Users feel scolded, not guided
2 Hierarchy High "3 things to review" is vague Show specific benefit: "Save $209/year" Users don't know what or why
3 Hierarchy High All items appear at equal priority Color-code by urgency: red / amber / blue Critical items hidden in flat list
4 Hierarchy Medium Yellow background suggests caution Use calm white, blue, or green backgrounds Feels like a system alert, not guidance
5 Clarity High "Add savings >" — action-first, benefit-last Lead with benefit: "Find idle cash" Users don't know why they should act
6 Clarity High No progress indication shown Add progress bar or counter (e.g. 3 of 7) Users don't know completion status
7 Clarity Medium "This is all of them" is confusing Replace with "I've connected all accounts" Ambiguous meaning — all of what?
8 Clarity Medium "Add an account" doesn't specify type Show account type in CTA text Users guess what to add
9 Clarity Medium Long descriptions overflow or wrap poorly Truncate with expandable "Read more" Text overflow on small screens
10 Clarity Low No time estimate per action Add "2 minutes" label on each card Users don't know time commitment
11 Clarity Low No benefit explanation given Add "What you'll gain" section per card No motivation to act
12 Interaction High "Add savings >" styled as text link, not button Use proper primary button styling Low tap confidence — looks tappable?
13 Interaction High No loading state on interaction Add skeleton shimmer or spinner on CTA Users tap multiple times causing duplicates
14 Interaction High No success state after completing action Add green checkmark animation on success No completion feedback — did it work?
15 Interaction Medium No dismiss or snooze option per card Add X button or snooze per recommendation Can't hide irrelevant items
16 Interaction Medium No expand/collapse — all info shown at once Add progressive disclosure pattern All info shown at once overwhelms users
17 Trust High No data source attribution on cards Show "Based on Chase data" or similar Users don't trust unattributed recommendations
18 Trust Medium No timestamp on data Show "Updated 2 min ago" on each card Stale data concerns erode confidence
19 Trust Medium Warning language feels like errors Use advisor tone: "I noticed..." phrasing Feels like system errors, not guidance
20 Trust Low No "why this matters" explanation Add expandable explanation per card Users skip recommendations without context
21 Scalability Medium Fixed layout — no support for long text Use Auto Layout with truncation rules Breaks on long account names
22 Scalability Medium No compact or expanded variant designs Design both compact and expanded sizes Same card used for all contexts
23 Scalability Low No priority level system Add critical / high / medium / low levels All items look the same importance
24 Accessibility High Status communicated by color only Add icons and patterns alongside color Colorblind users can't distinguish states
25 Accessibility Medium No VoiceOver labels on interactive elements Add aria-label to all interactive elements Screen reader users are lost

Section 2 — Component Behavior Specifications

8 states

Card States

Default Ready
White background, full opacity, ready for interaction. All CTAs are enabled and tappable.
Loading Skeleton
Skeleton shimmer animation, pulsing opacity. No interaction allowed during load.
Success Feedback
Green border flash with checkmark animation. Auto-dismisses after 2 seconds.
Error Recoverable
Red border with error message. Shows retry CTA to recover from failure.
Dismissed Reversible
Slides right and fades out over 200ms. Remains retrievable via undo action.
Expanded Detail
Height animates open over 200ms ease-out. Chevron icon rotates 180°.
Collapsed Compact
Height animates closed over 150ms ease-in. Returns to default compact view.
Disabled Inactive
50% opacity, no touch events, grayed-out text and CTAs. Unavailable state.

State Transitions

Default → Loading On user action (immediate)
Loading → Success API response success
Loading → Error API response failure
Default → Expanded Tap chevron / card body
Expanded → Default Tap again / collapse
Default → Dismissed Swipe right / dismiss CTA
Dismissed → Default Undo action

Section 5 — Accessibility Specifications

4 areas

VoiceOver Labels

Cards aria-label = [title] + [hint] — e.g. "Save $209 on subscriptions — double tap to expand"
CTAs aria-label = [text] + [action] — e.g. "Connect account — opens Chase login"
Dismiss aria-label="Dismiss" + hint — e.g. "Dismiss — swipe right to undo"
Checkboxes aria-label="Complete" / "Incomplete" with state toggle
Loading aria-live="polite" — "Loading recommendations"
Success aria-live="assertive" — "Completed: account connected"
Error aria-live="assertive" — "Error: [description]. Retry available."

Dynamic Type

Scale Support All text respects system Dynamic Type settings
Layout Reflow Layouts reflow up to 200% text size without breaking
No Truncation Critical information never truncated — CTAs, titles, errors
Button Resize Buttons resize to accommodate larger text sizes

Contrast Ratios

Normal Text Minimum 4.5:1 contrast ratio against background
Large Text (18pt+) Minimum 3:1 contrast ratio against background
Interactive Elements Minimum 3:1 ratio against adjacent colors
Focus Indicator 2px solid #0a84ff with 2px offset — visible on all backgrounds

Touch Targets

Minimum Size 44 × 44 points — per Apple HIG and WCAG 2.5.8
Spacing 8pt between adjacent targets — prevent mis-taps
No Overlap Touch regions never overlap or stack visually
Active State All tappable elements show pressed state feedback

Card Copy Formula

1
Headline
Lead with the benefit — what the user gains. Keep under 6 words.
2
Description
Explain why in one sentence. Provide context for the recommendation.
3
CTA
Action-oriented verb. Tell the user exactly what to do next.
4
Secondary / Time
Time estimate or secondary info. Set expectations for effort required.
