Usage Guidelines

  • Use Critical for account disconnections blocking core features
  • Use High for financial warnings (shortages, overdue items)
  • Use Medium for informational alerts requiring user awareness
  • Use Info for non-urgent updates that enhance context
  • Don't stack more than 2 alerts simultaneously
  • Don't use Critical for non-financial system messages
  • Don't auto-dismiss critical or high alerts

Accessibility

WCAG 2.1 Requirements • Role: alert for critical/high, status for medium/info
• aria-live: "assertive" for critical, "polite" for others
• Dismiss button: aria-label="Dismiss alert"
• CTA: descriptive label, not just "Click here"
• Color contrast: all text passes AA (4.5:1 minimum)
• Priority badge: include screen reader text "Priority: Critical"

Usage Guidelines

  • Match skeleton shape to actual content shape
  • Use skeleton only for initial page load
  • Replace skeleton with content as soon as data arrives
  • Don't show skeleton for small inline updates
  • Don't mix skeleton with actual content on same card
  • Don't use skeleton longer than 3 seconds — show error state after

Usage Guidelines

  • Use confirmation modals for destructive or irreversible actions
  • Use detail modals for complex information not fitting in a card

  • Don't nest modals (no modal inside modal)
  • Don't use modals for simple notifications — use toasts

Accessibility

• Backdrop: click to close (with aria-label="Close dialog")
• Title: use heading element (h2) linked via aria-labelledby
• Handle: aria-hidden="true" (decorative only)
