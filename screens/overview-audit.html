Hierarchy Issues

6 issues
#1 Welcome greeting wastes prime screen real estate Critical
Current State
"Welcome, James" hero banner occupies the entire first viewport. No financial data is visible until the user scrolls past the greeting.
Problem
Returning users already know their name. The greeting adds zero value while pushing critical financial data below the fold. First-screen real estate is wasted on decorative content.
Solution
Replace the greeting with a compact summary bar: net worth, cash available, and a single "Needs attention" count. Personalize with first name only. Keep greeting height under 56px.
Component
FinancialSummaryBar
#2 "3 things to fix" is ambiguous language Critical
Current State
Section header reads "3 things to fix" with no explanation of what "fix" means or what the consequences of inaction are.
Problem
Users don't know if "fix" means optimize, repair, or investigate. The term is vague and doesn't communicate urgency. Users may skip this section entirely.
Solution
Use specific action language: "Action required" for blockers, "Optimization opportunities" for improvements, "Needs review" for informational items. Include impact estimate per item.
Component
ActionItemList
#3 Optimization Score (68/100) is abstract and unactionable Medium
Current State
"Optimization Score 68/100" is displayed as a circular gauge with no explanation of what the score measures or how to improve it.
Problem
68 feels like a grade — it implies failure. Users don't know if this is good, bad, or average. There's no link between the score and specific actions. It creates anxiety without providing utility.
Solution
Replace abstract score with concrete health indicators: "3 accounts synced", "1 subscription overpaying", "0 goals off track". Each indicator links to an actionable item. Show score only if benchmarked against peers.
Component
HealthIndicators
#4 "Potential gain +$2,340/yr" is buried below other content Medium
Current State
The primary value proposition — "$2,340/yr potential gain" — is placed below the optimization score and action items, partially hidden.
Problem
This is the single most compelling number on the dashboard. It answers "why should I care?" yet it's visually de-emphasized. Users may never see it.
Solution
Move to top-right of summary bar as a persistent, prominent callout. Use green color and large type. Make it tappable to show breakdown. This is the user's motivation — keep it visible at all times.
Component
ValuePropositionBadge
#5 Money Flow duplicates information from Accounts section Medium
Current State
Money Flow shows inflows/outflows that already appear in the Accounts and Transactions sections. Same data presented twice.
Problem
Redundancy increases cognitive load. Users must decide which source to trust. Extra scroll distance without adding new information.
Solution
Replace Money Flow with a unique "Cash Flow Trend" that shows month-over-month change with trend arrows. Link to full transaction history. Show only net flow, not duplicate individual transactions.
Component
CashFlowTrend
#6 No answer to "What should I do next?" Critical
Current State
The dashboard presents data in sections but never tells the user what action to take. No primary CTA, no recommended next step.
Problem
The most important question for any financial dashboard — "what do I do now?" — goes unanswered. Users leave without taking action, reducing engagement and perceived value.
Solution
Add a persistent "Next step" card at top of dashboard. Dynamically prioritize: bill due today > optimization opportunity > account sync needed. Single prominent CTA per state. Never show more than 2 items.

NextStepCard

Scalability Issues

6 issues
#7 Account cards overflow with long institution names Critical
Current State
"Capital One Business Premium Checking" overflows the card boundary, truncating or breaking layout. No text-wrapping strategy is implemented.
Problem
Real bank names are often long. The current card assumes short names, causing overflow, truncation, or horizontal scroll. Looks broken on real data.
Solution
Use CSS text-overflow: ellipsis with max 2 lines. Show institution logo + short name (e.g., "Capital One"). Full name on hover/long-press. Card should handle names up to 40 characters gracefully.
Component
AccountCard
#8 Amount fields don't handle large numbers Critical
Current State
Move Money screen shows "$310343350.00" — raw float without formatting, commas, or abbreviation.
Problem
Unformatted numbers are unreadable and break layout. Users can't parse $310343350.00 at a glance. This is a fundamental data display bug.
Solution
Always format currency: use Intl.NumberFormat with locale-appropriate commas. For display-only contexts, abbreviate: "$310.3M". For input fields, allow raw entry but validate format. Show formatted version on blur.
Component
FormattedCurrency
#9 Portfolio section only shows 3 ETFs — no scrolling for 10+ holdings Medium
Current State
Portfolio section renders exactly 3 ETF items in a fixed layout with no overflow handling or "view more" mechanism.
Problem
Most portfolios have 5–20+ holdings. Users with more than 3 ETFs see no additional data. No indication that more items exist. Layout breaks or clips.
Solution
Show top 3 by allocation with "+N more" badge. Use horizontal scroll with snap for mobile. Add "View portfolio" link. Collapse to 2-line summary on mobile with expandable full list.
Component
HoldingsPreview
#10 Spending section only shows 3 categories Medium
Current State
Spending breakdown displays only 3 categories (Food, Transport, Shopping) in a fixed list with no expansion mechanism.
Problem
Real spending has 10–20+ categories. Users can't see Housing, Utilities, Subscriptions, etc. The summary is incomplete and misleading.
Solution
Show top 3 categories with horizontal bar chart. "View all spending" link below. On tap, expand to full list or navigate to Spending tab. Show total at bottom of preview.
Component
SpendingPreview
#11 News section is static — no empty state or overflow handling Medium
Current State
News section always shows 3 hardcoded articles. No mechanism for 0 news items or 20+ items.
Problem
New users see stale content. Power users see only 3 items. No indication of freshness. Section doesn't adapt to data availability.
Solution
Show "No news yet" empty state for new users. Show 3 most recent with "View all" link. Show "Updated X min ago" timestamp. If 0 relevant items, hide section entirely.
Component
NewsFeed
#12 No responsive behavior for different screen sizes Critical
Current State
Layout is fixed-width. No breakpoints. On mobile, content overflows horizontally or becomes unreadably small.
Problem
Dashboard will be used on phones, tablets, and desktops. Fixed layout breaks on smaller screens. Horizontal scroll on mobile is a critical usability failure.
Solution
Implement responsive grid: 1 column on mobile, 2 on tablet, 3 on desktop. Use max-width: 960px container. Stack cards vertically on mobile. Hide secondary sections below the fold on small screens.

Onboarding Issues

5 issues
#13 No loading states — dashboard appears instantly Medium
Current State
The entire dashboard renders at once with no skeleton screens, spinners, or progressive loading indicators.
Problem
Unrealistic — real data takes time to fetch. Users see stale data or flash of content. No feedback that the app is working. Broken trust if data loads incorrectly.
Solution
Show skeleton screens per section while data loads. Use shimmer animation on placeholder cards. Load summary bar first, then sections in priority order. Show "Syncing accounts..." status.
Component
SkeletonLoader
#14 No empty states for new users Critical
Current State
Dashboard shows full data as if user has been using the app for months. No empty states for new accounts.
Problem
New users see confusing placeholder data or broken states. No guidance on what to do first. High drop-off at first session — users don't know the app needs setup.
Solution
Design 3 empty states: (1) No accounts connected — "Connect your first account", (2) Accounts connected, no data — "Syncing...", (3) Data ready — "Here's your overview". Each with a clear CTA.
Component
EmptyState
#15 No progressive disclosure — everything shown at once Medium
Current State
All sections — Accounts, Portfolio, Spending, News, Action Items — are visible simultaneously regardless of user's familiarity or financial complexity.
Problem
Overwhelms new users with information they don't need yet. Hides advanced features from power users. No adaptation to user behavior or financial complexity.
Solution
Start with 3 core sections: Summary, Action Items, Next Step. Add Portfolio, Spending, and News as user completes onboarding. Use "Customize dashboard" to let users reorder/hide sections.
Component
ProgressiveDisclosure
#16 No onboarding flow — user dropped into full dashboard Critical
Current State
After signup, user immediately sees the complete dashboard with all sections populated. No guided walkthrough.
Problem
Users don't understand what they're looking at. No context for the data. No guidance on connecting accounts. First-session experience is confusing and impersonal.
Solution
3-step onboarding: (1) "Connect your first account" — bank selection, (2) "Here's your snapshot" — highlight 3 key data points, (3) "What matters most?" — let user choose priorities. Total time under 60 seconds.
Component
OnboardingWizard
#17 No setup progress indicator Medium
Current State
No visible indication of what setup tasks are complete or remaining. User doesn't know if their dashboard is fully configured.
Problem
Users assume the dashboard is complete after first login. They don't know they can connect more accounts or set up goals. Reduces feature adoption.
Solution
Add collapsible progress bar below summary: "Setup: 3/5 complete". Checklist items: Connect accounts (✓), Set goals (✓), Connect investments ( ), Link credit cards ( ), Set budget ( ). Auto-dismiss when 100%.

Interaction Issues

5 issues
#18 All sections are static cards — no expandable content Medium
Current State
Every section is a flat card with fixed content. No expand/collapse, no accordion behavior, no progressive disclosure.
Problem
Dashboard feels like a printout, not an interactive tool. Users can't drill into details without leaving the page. No way to reduce information density.
Solution
Add chevron-expand on each section header. Default state: collapsed (showing summary). Tap to expand full content. Persist user's expand/collapse preferences. Animate height transitions.
Component
ExpandableSection
#19 "Review & fix all" is the only interaction — no inline actions Critical
Current State
Action items list has a single "Review & fix all" button at the bottom. Individual items have no tappable actions.
Problem
Users must review all items even if they only want to fix one. No way to dismiss irrelevant items. Forces a linear workflow that doesn't match real behavior.
Solution
Add inline actions per item: "Fix now" (primary), "Dismiss" (secondary), "Learn more" (info). Keep "Fix all" as bulk action. Allow swipe-to-dismiss on mobile. Show item count badge on button.
Component
ActionItem
#20 No pull-to-refresh — user can't manually sync accounts Low
Current State
No manual refresh mechanism. Data syncs only on app open. User has no way to trigger an immediate sync.
Problem
After making a transaction, user expects to see updated balance. No way to force refresh. Feels broken when data is stale. Standard mobile pattern missing.
Solution
Add pull-to-refresh on mobile. Show "Last synced 2 min ago" indicator. Manual "Sync now" button in header. Show sync progress animation during refresh.
Component
PullToRefresh
#21 No haptic feedback indicators Low
Current State
All interactive elements look identical to static content. No visual or tactile cues for tappability.
Problem
Users can't distinguish between content and buttons. Important actions go untapped. Static feel reduces engagement.
Solution
Add hover states (desktop), press states (mobile), and subtle elevation changes for tappable items. Use consistent tap targets (min 44px). Add haptic feedback on supported devices.

InteractiveFeedback
#22 No transition animations — content appears/disappears instantly Low
Current State
Sections render instantly with no fade-in, slide, or morph animations. State changes are abrupt.
Problem
Feels jarring and unpolished. Users lose spatial context when content jumps. No visual continuity between states.
Solution
Add 200ms ease-out transitions for expand/collapse. Fade-in on initial load with staggered delays per section. Animate number counters on data updates. Use prefers-reduced-motion to disable for accessibility.

Trust & Transparency Issues

3 issues
#23 No "Last synced" indicator Critical
Current State
No timestamp or indicator showing when account data was last refreshed from financial institutions.
Problem
Users can't tell if data is current or days old. Making financial decisions on stale data is dangerous. No trust in the numbers shown.
Solution
Add persistent "Last synced: 2 min ago" in header. Color-code: green (< 1 hr), amber (1–24 hr), red (> 24 hr). Tap to see per-account sync status. Show sync errors inline.
Component
SyncStatus
#24 No connection status for linked accounts Critical
Current State
Account cards show name and balance but no indication of connection health — whether the bank link is active, expired, or broken.
Problem
Users don't know if data is being actively synced. Expired connections silently fail. Balances appear correct but are actually stale. No way to proactively fix issues.
Solution
Add status dot per account: green (connected), amber (re-auth needed), red (error). Show "Reconnect" button inline when needed. Batch re-authentication for multiple expired connections.
Component
AccountConnectionStatus
#25 No error handling for failed bank connections Critical
Current State
No visible error states for failed connections, sync failures, or data inconsistencies. Errors are silently swallowed or shown as empty sections.
Problem
Users think the app is broken. No way to know what went wrong or how to fix it. Support tickets increase. Users lose trust and churn.
Solution
Show inline error cards per section: "Could not sync Chase — tap to retry". Global error banner for system-wide issues. Error states with specific recovery actions. Never show empty data as "normal".
Component
ErrorState
