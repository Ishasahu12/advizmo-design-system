UX Audit — 36 Identified Issues

Comprehensive audit of the current Advizmo overview experience. Each issue is categorized, severity-rated, and paired with a recommended fix and UX rationale.

Hierarchy

#SeverityCurrent StateRecommended FixUX Reason
1 High Net Worth displayed as primary KPI but doesn't reflect actionable insight Replace with "Available Cash" or "Monthly Surplus" as primary metric; move Net Worth to Portfolio section Users can't act on Net Worth — it's vanity. Actionable metrics drive better financial decisions
2 High Account balances shown without context (no trend, no comparison) Add mini trend line or "% change vs last month" below each balance A raw number means nothing without temporal context — users need to know direction
3 High Health Score displayed without explanation of what it measures Add tooltip icon explaining "Based on savings rate, debt ratio, and emergency fund" with breakdown Unexplained scores erode trust. Users must understand how the score is derived
4 Medium Portfolio section uses "Net Liquidation Value" — financial jargon Use "Portfolio Value" or "Investment Value" with a "What's this?" tooltip Jargon alienates 60%+ of users who aren't finance professionals
5 Medium Spending section mixes categories without clear hierarchy Sort by spend amount descending, group into "Essentials" / "Discretionary" / "Other" Users scan top-down — highest spend must be most visible for budget awareness
6 Medium News section competes with financial data for attention Move News below the fold, reduce card height, or collapse into "Insights" summary News is informational — it should never compete with actionable financial data
7 High No visual distinction between "needs attention" and "informational" Use colored left-border accents: red for attention, amber for caution, blue for info Users need to instantly know what requires action vs. what's passive information

Scanning

#SeverityCurrent StateRecommended FixUX Reason
8 Critical Too many sections visible at once — cognitive overload Limit to 3 sections visible, collapse rest with "Show More" or horizontal scroll Miller's Law: users can process ~7±2 items. Too many = none get attention
9 Medium No visual anchors for eye tracking Add colored section headers, icons, or numbered badges to each section Visual anchors reduce search time by 30-40% in dense layouts
10 Low Account names truncate on long values Use ellipsis with tooltip on long names, max 24 chars displayed Truncated text without context is frustrating — users can't identify accounts
11 High No clear scan path from most to least important Establish hierarchy: Alerts → KPIs → Accounts → Actions → Information Without scan path, users wander and miss critical information
12 Medium Mixed card styles reduce scannability Standardize all cards: 14px radius, 16px padding, consistent shadow Visual consistency reduces cognitive load and speeds recognition

Accessibility

#SeverityCurrent StateRecommended FixUX Reason
13 Critical Color-only status indicators (no icons/patterns) Add icon badges alongside color: ✓ green, ⚠ amber, ✕ red, ℹ blue 8% of men are color blind — color alone is insufficient for 1 in 12 users
14 High Touch targets below 44x44pt on some interactive elements Increase all tap targets to 44x44pt minimum with adequate spacing Apple HIG requires 44pt minimum — smaller targets cause mis-taps
15 Medium No Dynamic Type support indicators Use relative units (rem/em), test at AX5 size, add Dynamic Type preview Users with visual impairments rely on larger text — ignoring this excludes them
16 High Missing VoiceOver labels on charts Add accessibilityLabel with summary: "Spending chart: $3,200 total, highest: Dining" Blind users can't interpret visual charts without text alternatives
17 Medium No reduced motion alternatives Respect prefers-reduced-motion: disable skeleton pulse, use opacity fade instead Vestibular disorders affect 35% of adults over 40 — motion can cause nausea

Empty States & Loading

#SeverityCurrent StateRecommended FixUX Reason
18 Critical No empty state design for first-time users Create illustrated empty states with clear CTAs and expected benefits First-time experience sets expectations — blank screens cause abandonment
19 High No skeleton loading — shows blank then jumps Implement skeleton placeholders that pulse during data fetch Skeletons reduce perceived load time by 15-20% and prevent layout shift
20 Medium No progressive disclosure during onboarding Show 2-3 sections first, reveal others as user connects accounts Progressive disclosure reduces overwhelm and guides users naturally
21 Medium No "what to do next" guidance Add contextual CTA banners: "Connect your checking to see your balance" Users need direction — ambiguous next steps cause drop-off

Loading & Onboarding

#SeverityCurrent StateRecommended FixUX Reason
22 High Full-screen spinner on refresh Use section-level skeletons with pull-to-refresh indicator Full-screen spinners feel broken — skeleton states feel like progress
23 Medium No section-level loading indicators Add small spinner or skeleton per section as it loads independently Sections load at different speeds — show progress per section
24 Low No optimistic UI for common actions Show success state immediately, reconcile in background Optimistic UI makes actions feel instant — reduces perceived latency
25 Medium No setup progress indicator Add persistent progress bar: "Financial Setup — 3/7 Completed" Visible progress motivates completion — the Zeigarnik effect
26 Low No contextual help for first-time features Add tooltip popovers on first visit with "Got it" dismissal Contextual help reduces support tickets and improves feature adoption
27 Medium No "why connect this" explanations Add benefit text: "Connect checking to track spending patterns" Users won't connect accounts without understanding the value proposition

Progressive Disclosure & Financial Trust

#SeverityCurrent StateRecommended FixUX Reason
28 High All information shown at once — no drill-down Add tap-to-expand on each section with detailed breakdown Progressive disclosure prevents overwhelm while keeping info accessible
29 Medium No expand/collapse for sections Add chevron toggle on section headers, persist collapsed state Users should control information density based on their needs
30 Low No "learn more" for complex concepts Add inline "Learn more" links that open bottom sheet explanations Financial concepts need explanation — tooltips aren't enough for complex topics
31 Critical No timestamp for "last synced" Show "Last synced: 2 minutes ago" with manual refresh option Financial data staleness destroys trust — users must know data freshness
32 High No data source attribution Show "via Plaid" or bank logo next to each account Users need to know where data comes from to trust it
33 Medium No disclaimer for projections Add "Projections are estimates based on historical data" footnote Financial projections without disclaimers create liability and false confidence
34 High No offline handling Show offline banner, display cached data, queue actions for sync Fintech apps must degrade gracefully — blank screens on offline are unacceptable
35 Critical No connection error recovery Show per-account error banners with "Reconnect" CTA and retry logic Connection failures are inevitable — recovery UX is critical for retention
36 Medium No handling of zero balances Show "$0.00" with contextual message: "No transactions yet" or "Account closed" Zero is ambiguous — users need to know if it's intentional or an error

User Journey — 12 Stages

The dashboard evolves as users connect accounts and configure their financial profile. Each stage shows different content, alerts, and CTAs.

Stage 1

Brand New

Connected: Nothing
Missing: All accounts
Dashboard: Welcome hero, setup progress (0/10), empty sections with CTAs
Alerts: "Welcome to Advizmo — let's get started"
CTAs: "Connect Your First Account" (primary)
Stage 2

Checking Connected

Connected: Checking only
Missing: Savings, investments
Dashboard: Checking balance, setup progress 1/4, empty portfolio/spending
Alerts: "Connect Savings to see your full cash picture" (High)
CTAs: "Connect Savings" (primary), "Add Investment Account" (secondary)
Stage 3

Checking + Savings

Connected: Checking, Savings
Missing: Investments, credit cards
Dashboard: Both balances, cash flow emerging, basic spending
Alerts: "Connect Investment accounts for full net worth" (Medium)
CTAs: "Connect Investments" (primary)
Stage 4

External Connected, No Advizmo Account

Connected: All external accounts
Missing: Advizmo managed account
Dashboard: Full financial picture, net worth visible, spending breakdown
Alerts: "Create an Advizmo Account to earn 4.25% APY" (High)
CTAs: "Create Advizmo Account" (primary)
Stage 5

Advizmo Account Created

Connected: All accounts + Advizmo
Missing: Recurring review
Dashboard: Accounts visible, recurring section empty with CTA
Alerts: "Review your recurring transactions for changes" (High)
CTAs: "Review Recurring Transactions" (primary)
Stage 6

Recurring Review Complete

Connected: All accounts
Missing: Profile completion
Dashboard: Clean dashboard, recurring items displayed, health score visible
Alerts: "Complete your profile for personalized insights" (Medium)
CTAs: "Complete Profile" (primary)
Stage 7

Targets Missing

Connected: All accounts, profile done
Missing: Financial targets
Dashboard: Most sections healthy, targets section empty
Alerts: "Set financial targets to track your goals" (Medium)
CTAs: "Set Financial Targets" (primary)
Stage 8

Cushions Missing

Connected: All accounts, targets set
Missing: Cushion configuration
Dashboard: Almost complete, cushions section shows placeholder
Alerts: "Configure cushions for financial flexibility" (Medium)
CTAs: "Configure Cushions" (primary)
Stage 9

Healthy Dashboard

Connected: Everything configured
Missing: Nothing
Dashboard: Full, healthy, all green sections, recommendations visible
Alerts: None (or low-priority recommendations)
CTAs: "View Recommendations" (secondary)
Stage 10

Needs Attention

Connected: All accounts
Issue: Something wrong (low balance, exceeded budget, etc.)
Dashboard: Alert banner at top, affected section highlighted red
Alerts: Critical or High alert with specific CTA
CTAs: "Fix Now" (primary) on affected section
Stage 11

Offline

Connected: All accounts
Issue: No internet connection
Dashboard: Offline banner, cached data with "last updated" timestamps
Alerts: "You're offline. Showing cached data." (System banner)
CTAs: "Retry Connection" (primary)
Stage 12

Connection Errors

Connected: Some accounts, some errored
Issue: Bank API down or credentials expired
Dashboard: Error banners per affected account, partial data visible
Alerts: "Chase needs re-verification" per account (Critical)
CTAs: "Reconnect Chase" (primary), "Dismiss" (secondary)

Display Rules
Critical: Always shown at top. Dismissed only by user action or resolution.
High: Max 3 visible. Oldest auto-collapses after 3 are shown.
Medium: Max 3 visible. Collapsed into "View 2 more alerts" after limit.
Info: Collapsed by default. Shows as "💡 1 tip available" expandable link.
Ordering: Critical → High → Medium → Info. Within priority: newest first.
Dismissal: Dismissed alerts move to "Alert History" in settings.

Loading States

Skeleton components for every section. Never show blank screens — always communicate loading progress.

Skeleton KPI Card
Skeleton Account Card
Skeleton Portfolio
Skeleton Spending
Loading Principles
1. Never use full-screen spinners — always use skeleton placeholders
2. Show skeletons immediately (0ms delay) — no blank frames
3. Progressive reveal: Accounts load first → Portfolio → Spending → News
4. Each section has its own independent loading indicator
5. Show "Last synced: 2 minutes ago" during load to maintain trust
6. Skeleton pulse animation: 1.5s infinite, ease-in-out
7. Skeleton color: #e5e5ea base, #d1d1d6 highlight

Interactions

All animation and interaction specifications with timing, easing, and conditions.

Expand
200ms ease-out, height auto + opacity 0→1, overflow hidden during transition
Collapse
150ms ease-in, height to 0 + opacity 1→0, overflow hidden during transition
Loading Skeleton
Pulse animation 1.5s infinite, ease-in-out, background-position shift
Pull-to-Refresh
Pull indicator at top, 300ms spin animation on release, haptic feedback
Dismiss
Slide right 100% + fade, 200ms ease-in, haptic feedback on complete
Swipe
Horizontal, 50px velocity threshold, reveals action buttons underneath
Success
Green checkmark fade-in + scale, 300ms ease-out, haptic "success" pattern
Failure
Red X shake (3px left-right), 200ms, haptic "error" pattern
Retry
Button shows spinner, disabled state, re-enables on completion or timeout
Reduced Motion
All animations replaced with instant opacity toggle, no movement
Interaction Principles
1. Every interaction provides visual feedback within 100ms
2. Animations never block user input — content updates behind animation
3. All animations respect prefers-reduced-motion media query
4. Haptic feedback accompanies all success/failure states (iOS)
5. Touch targets minimum 44x44pt with 8pt spacing between

Accessibility

Complete accessibility specification ensuring WCAG AA compliance and iOS accessibility standards.

📐 Dynamic Type

All text respects iOS Dynamic Type settings. Use relative units (rem/em). Test at AX1 through AX5 sizes. Layout reflows without horizontal scrolling at any text size.

🔊 VoiceOver

Every interactive element has accessibilityLabel + accessibilityHint. Charts provide summary labels. Images have descriptive alt text. Reading order follows visual hierarchy.

👆 Touch Targets

Minimum 44x44pt on all interactive elements. 8pt minimum spacing between adjacent targets. Larger targets for frequently used actions (CTAs, navigation).

🎭 Reduced Motion

All animations respect prefers-reduced-motion. Skeleton pulse replaced with static placeholder. Transitions replaced with instant opacity changes. No auto-playing animations.

🎨 Color Contrast

WCAG AA minimum: 4.5:1 for normal text, 3:1 for large text (18px+). All status colors tested against their backgrounds. Never use color alone to convey information.

📢 Loading Announcements

VoiceOver announces: "Loading your accounts" when data fetch begins. "Accounts loaded" when complete. "Unable to connect to Chase" on error. "Account connected successfully" on success.

⌨️ Keyboard Navigation

All interactive elements focusable via tab. Focus ring visible (2px blue outline). Logical tab order follows visual layout. Escape key collapses expanded sections.

🧩 Semantic HTML

Use proper heading hierarchy (h1→h2→h3). Landmark regions for nav, main, aside. Lists for repeated items. Tables for tabular data with proper th/scope.
